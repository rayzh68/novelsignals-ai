# NovelSignals AI - Project Requirements

## Core Positioning

NovelSignals AI is a Novel Market Intelligence System.

The system focuses on:

- Public novel metadata collection
- Market signal discovery
- SEO asset generation
- Creative asset generation
- Structured content bundle export

The system does NOT build websites.

Website generation is handled by external tools:

- Codex
- Cursor
- Claude Code
- CMS Systems

NovelSignals AI outputs structured data packages only.

---

## Ranking Rules

Platform rankings must preserve original platform ordering.

Examples:

- GoodNovel Rankings
- Dreame Rankings
- WebNovel Rankings

NovelSignals rankings are independent rankings generated by NovelSignals Score.

Platform rankings and NovelSignals rankings must remain separate.

---

## Page Types

Supported page types:

- Home Page
- Book Detail Page
- Ranking Page
- Topic Page
- Book Comparison Page
- Platform Comparison Page

Avoid introducing new page types unless absolutely necessary.

---

## File Structure Principle

Minimize file count.

Minimize folder depth.

Prefer extending existing modules.

Avoid creating unnecessary scripts.

---

## Topic Rules

Topic pages should be generated from:

- Market Signals
- Catalog Analysis

Do NOT rely on hardcoded topic definitions.

---

## Scope Boundary

NovelSignals AI is responsible for:

- Discovery
- Metadata
- Catalog
- Market Signals
- SEO Assets
- Creative Assets
- Content Bundles

NovelSignals AI is NOT responsible for:

- Frontend implementation
- Website rendering
- CMS implementation

These are downstream systems.
